    Git є критично важливим інструментом у сучасних проєктах, оскільки він забезпечує надійний контроль версій, дозволяє відстежувати кожну зміну коду та робить
можливою ефективну колективну розробку. Мій особистий досвід після виконання завдання показав, що найбільше зусиль потребувала початкова конфігурація середовища 
(налаштування SSH-зв'язку та змінної PATH для g++).
    Найзручнішим виявився механізм гілок (branching), що дозволяє безпечно ізолювати розробку нового функціоналу у гілці develop. Це дало змогу спокійно вносити 
зміни до README.md та логіки калькулятора, не ламаючи при цьому основну гілку master. Найважчим було коректне об'єднання (merge) змін з віддаленого репозиторію 
особливо після того, як я об'єднав Pull Request на GitHub і мій локальний master відстав.
    Навички роботи з Git, зокрема, практика використання гілок і Pull Request, будуть обов'язково застосовані у курсовому проєкті для безпечної інтеграції різних 
модулів та ефективної співпраці в команді, якщо така буде.
